Git/Github

Git er et distribuert versjonskontrollsystem for sporing av versjoner av filer. Versjonskontroll er et system som registrerer endringer i en fil eller et sett med filer over tid. Github er webportal for Git-repositoriene. Git lar deg spore og vert versjoner av filer på Github. Med andre ord bruker du kommandoer til Git til å spore versjoner av filene dine og Github er bare en ekstern plattform hvor disse filene er vert. Github hjelper utviklere for å samarbeide sammen offentlig eller privat.
GitHub-flow er en lett og grenbasert bygget rundt Git-kommandoer brukt av lag over hele verden. 

      GitHub-flow har seks trinner
       -Lag en branch
       -Add commits
       -Open a pull request
       -Discuss and review code
       -Merge
       -Deploy
  • Repository: er en mappe som du har lagt i Git for å spore filendringer.
  • Branch: En branch er en uavhengig utviklingslinje. Du kan tenke på det som en helt ny arbeidskatalog.
  • Fork: En fork er en personlig kopi av en annen brukers repository som lever på kontoen din.
  • clone: En clone er bare en kopi av et lager som ligger på datamaskinen din i stedet for på en server.
  • Commit: En commit er et sett med en eller flere endringer i en fil eller et sett med filer. Hver gang du lagrer, oppretter den en unik ID som hjelper den med å holde oversikt over historien.
  • Master: er en standard utviklingsgren. Når du oppretter en git repository, opprettes en branch som heter "master" som blir den aktive grenen.
